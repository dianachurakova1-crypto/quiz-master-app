# Quiz Master | Веб-приложение для квизов в реальном времени

Платформа для создания и проведения квизов с поддержкой комнат, WebSocket и автоматическим подсчётом баллов.

---

## Что умеет приложение

**Пользователи.** Есть регистрация и вход. При регистрации нужно указать никнейм, электронную почту и пароль. При входе — только почту и пароль. Пароль можно показывать или скрывать кнопкой-глазиком.

**Создание квизов.** Организатор создаёт квиз, добавляет вопросы. К каждому вопросу можно добавить текст, картинку, варианты ответов. На выбор — одиночный или множественный выбор ответа. Для каждого вопроса настраивается количество баллов и время на ответ.

**Проведение игры.** Организатор запускает квиз и получает уникальный код комнаты. Участники вводят этот код и своё имя, после чего подключаются к игре. Все вопросы показываются одновременно всем участникам, а ответить можно только пока идёт таймер.

**Подсчёт баллов и победители.** Система автоматически считает баллы за правильные ответы. Во время игры обновляется таблица лидеров. После последнего вопроса объявляется победитель. Если несколько игроков набрали одинаковое максимальное количество баллов — победителей несколько.

**Личный кабинет.** В личном кабинете можно посмотреть список созданных квизов, историю участия в чужих квизах, а также историю проведённых игр с указанием даты, победителя и количества участников.

---

## Как это работает внутри

Приложение состоит из серверной части на Node.js и клиентской части — обычный HTML, CSS и JavaScript. Весь код лежит в одном репозитории.

**Сервер.** Написан на Node.js с использованием фреймворка Express. Он обрабатывает запросы, раздаёт статические файлы и управляет WebSocket-соединениями через библиотеку Socket.IO.

**Хранилище.** В текущей версии данные хранятся в оперативной памяти (JavaScript-объекты Map). Это сделано для простоты запуска — не нужно устанавливать и настраивать базу данных. При необходимости можно легко заменить на PostgreSQL или MongoDB.

**Клиент.** HTML-страницы, CSS-стили и JavaScript-код лежат в папке public. JavaScript отвечает за подключение к серверу через Socket.IO, отрисовку интерфейса и обработку действий пользователя.

**Обмен данными.** Сервер и клиент общаются через WebSocket. Когда организатор начинает вопрос, сервер рассылает его всем участникам. Когда участник отвечает, сервер проверяет ответ, начисляет баллы и обновляет таблицу лидеров у всех игроков.

## Почему выбраны именно эти технологии

**Node.js и Express.** Весь проект написан на JavaScript — и сервер, и клиент. Это упрощает разработку, не нужно переключаться между языками. Node.js хорошо подходит для приложений с большим количеством одновременных соединений, что важно для квизов.

**Socket.IO.** Эта библиотека обеспечивает надёжное real-time соединение между сервером и клиентами. Она автоматически переподключается при обрыве связи и умеет создавать комнаты — идеально для разделения игроков по разным квизам.

**Vanilla JavaScript.** Отказ от фреймворков вроде React сделан осознанно. Приложение небольшое, и лишние зависимости только замедлили бы его работу. На чистом JavaScript проще отлаживать код, а итоговый размер файлов меньше.

**In-Memory хранилище.** На этапе прототипа важно быстро проверить работу логики, а не настраивать базу данных. In-Memory позволяет запустить приложение одной командой без дополнительных сервисов. Код написан так, что замена на реальную базу данных займёт не больше дня.

**Зелёная цветовая гамма.** Зелёный цвет ассоциируется с правильными ответами, успехом и спокойствием. Интерфейс не напрягает глаза, а акценты (правильный ответ — зелёный, неправильный — красный) интуитивно понятны.

## Как запустить

```bash
git clone https://github.com/dianachurakova1-crypto/quiz-master-app.git
cd quiz-master-app
npm install
node server.js
После этого откройте браузер и перейдите по адресу http://localhost:3000.

Как играть
Организатор:

Зарегистрируйтесь и войдите в систему.

Нажмите «Создать квиз» и добавьте вопросы с вариантами ответов.

Сохраните квиз, затем нажмите «Запустить» и введите код комнаты.

Разошлите код участникам и нажмите «Начать игру».

После каждого вопроса нажимайте «Следующий вопрос» — кнопка становится активной только после окончания таймера или когда ответят все игроки.

Участник:

Зарегистрируйтесь и войдите в систему.

Нажмите «Присоединиться к квизу».

Введите код комнаты и своё имя.

Отвечайте на вопросы, пока не закончится таймер.

После последнего вопроса все увидят победителя и финальную таблицу результатов.

Что особенного в этом проекте
Подсветка ответов. После того как таймер закончился, участники видят, какие ответы были правильными (зелёным), а какие они выбрали неправильно (красным).

Несколько победителей. Если два или больше игроков набрали одинаковое максимальное количество баллов, все они считаются победителями.

История проведений. У каждого созданного квиза есть кнопка «Проведённые». Нажав на неё, можно увидеть все запуски этого квиза с датой, победителем и количеством участников.

Кнопка «На главную». Она есть на всех экранах — в лобби, во время игры и на экране ожидания. Игрок в любой момент может выйти и вернуться в личный кабинет.

Защита от преждевременного переключения. Организатор не может переключить вопрос до того, как закончится таймер. Кнопка «Следующий вопрос» блокируется и становится активной только после сигнала от сервера.

Планы на будущее
Если развивать проект дальше, в первую очередь стоит заменить In-Memory хранилище на реальную базу данных — например, PostgreSQL или MongoDB. Это позволит сохранять данные между перезапусками сервера и масштабировать приложение.

Также можно добавить категории вопросов, чтобы игроки могли выбирать тематику квиза. Ещё один вариант — контейнеризация через Docker, чтобы упростить развёртывание на любом сервере.
