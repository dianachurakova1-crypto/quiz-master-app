# Vикторинс | Веб-приложение для квизов в реальном времени

Платформа для создания и проведения квизов с поддержкой комнат, WebSocket и автоматическим подсчётом баллов.

## Что умеет приложение

**Пользователи.** Есть регистрация и вход. При регистрации нужно указать никнейм, электронную почту и пароль. При входе — только почту и пароль. Пароль можно показывать или скрывать кнопкой-глазиком. Если пользователь уже регистрировался в текущем сеансе работы сервера, то его данные запоминаются - он может зайти только через вход, а не через новую регистрацию. 

**Создание квизов.** Организатор создаёт квиз, добавляет вопросы. К каждому вопросу можно добавить текст, картинку, варианты ответов. На выбор — одиночный или множественный выбор ответа. Для каждого вопроса настраивается количество баллов и время на ответ (таймер).

**Проведение игры.** Организатор запускает квиз и получает уникальный код комнаты (могут его придумать самостоятельно или скопировать случайно сгенерированный). Участники вводят этот код и своё имя, после чего подключаются к игре. Все вопросы показываются одновременно всем участникам, а ответить можно только пока идёт таймер.

**Подсчёт баллов и победители.** Система автоматически считает и суммирует баллы за правильные ответы. Во время игры обновляется таблица лидеров. После последнего вопроса объявляется победитель. Если несколько игроков набрали одинаковое максимальное количество баллов — победителей несколько.

**Личный кабинет.** В личном кабинете (на главном экране) можно посмотреть список созданных квизов, историю участия в чужих квизах, а также историю проведённых игр с указанием даты, победителя и количества участников.

## Как это работает внутри

Приложение состоит из серверной части на Node.js и клиентской части — HTML, CSS и JavaScript. Весь код лежит в одном репозитории.

**Сервер.** Написан на Node.js с использованием фреймворка Express. Он обрабатывает запросы, раздаёт статические файлы и управляет WebSocket-соединениями через библиотеку Socket.IO.

**Хранилище.** В текущей версии данные хранятся в оперативной памяти (JavaScript-объекты Map). Это сделано для простоты запуска. При необходимости можно заменить на PostgreSQL или MongoDB.

**Клиент.** HTML-страницы, CSS-стили и JavaScript-код лежат в папке public. JavaScript отвечает за подключение к серверу через Socket.IO, отрисовку интерфейса и обработку действий пользователя.

**Обмен данными.** Сервер и клиент общаются через WebSocket. Когда организатор начинает вопрос, сервер рассылает его всем участникам. Когда участник отвечает, сервер проверяет ответ, начисляет баллы и обновляет таблицу лидеров у всех игроков.

## Почему выбраны именно эти средства разработки

**Node.js и Express.** Весь проект написан на JavaScript — и сервер, и клиент. Это упрощает разработку, не нужно переключаться между языками. Node.js подходит для приложений с большим количеством одновременных соединений, одним из которых и является мое приложение для создания веб-квизов.

**Socket.IO.** Эта библиотека обеспечивает надёжное соединение между сервером и клиентами в реальном времени. Она автоматически переподключается при обрыве связи и умеет создавать комнаты.

**Vanilla JavaScript.** Отказ от фреймворков вроде React сделан осознанно. Приложение небольшое, и лишние зависимости только замедлили бы работу. Только на JavaScript проще отлаживать код, а итоговый размер файлов меньше.

**In-Memory хранилище.** На этапе сложная логика БД не нужна. In-Memory позволяет запустить приложение одной командой без дополнительных сервисов. На данном этапе замена на реальную БД не должна занять много времени.

## Как играть

### Организатор

1. Зарегистрируйтесь и войдите в систему.
2. Нажмите «Создать квиз» и добавьте вопросы с вариантами ответов.
3. Сохраните квиз, затем нажмите «Запустить» и введите код комнаты.
4. Разошлите код участникам и нажмите «Начать игру».
5. После каждого вопроса нажимайте «Следующий вопрос» — кнопка становится активной только после окончания таймера.

### Участник

1. Зарегистрируйтесь и войдите в систему.
2. Нажмите «Присоединиться к квизу».
3. Введите код комнаты и своё имя.
4. Отвечайте на вопросы, пока не закончится таймер.

После последнего вопроса все увидят победителя и финальную таблицу результатов. С этой страницы можно будет вернуть «На главную».

## Что ещё интересного добавлено в проект

**Подсветка ответов.** После того как таймер закончился, участники видят, какие ответы были правильными (зелёным), а какие они выбрали неправильно (красным).

**Несколько победителей.** Если два или больше игроков набрали одинаковое максимальное количество баллов, все они считаются победителями.

**Обратная связь.** Система постоянно выдаёт обратную связь — в правом нижнем углу при совершении значимых действий приложение сообщает, что сделал пользователь: неправильно ответил, удалил вопрос при создании квиза — кончился таймер и можно переходить к следующему вопросу и т. д.

**Кнопка «На главную»/«Выход».** Пользователь не заложник сервиса, в любой момент (кроме времени проведения квиза) он может вернуться на главный экран.

**Защита от преждевременного переключения.** Организатор не может переключить вопрос до того, как закончится таймер. Кнопка «Следующий вопрос» блокируется и становится активной только после сигнала от сервера.
